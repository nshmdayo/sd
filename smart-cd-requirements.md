# smart-cd 要件定義書

**ドキュメントID:** SCD-REQ-001  
**バージョン:** 1.0.0  
**作成日:** 2026-03-07  
**ステータス:** Draft  
**対象OS:** Linux / macOS (bash / zsh)  
**開発言語:** Go

---

## 1. 目的・背景

標準の `cd` コマンドはディレクトリの絶対パス・相対パスを毎回入力する必要があり、深いディレクトリ階層や頻繁な切り替えが発生する開発作業において生産性の低下を招く。

本ツール **smart-cd** は、インテリジェントなディレクトリ補完・ブックマーク管理・履歴ジャンプ機能を提供することで、ディレクトリナビゲーションを高速化・効率化することを目的とする。

---

## 2. スコープ

### 2.1 対象ユーザー

- Linux / macOS 環境で bash または zsh を使用するエンジニア・開発者
- 複数プロジェクトを並行して扱い、ディレクトリ移動が多い作業者

### 2.2 対象外

- Windows (PowerShell / CMD) — 将来バージョンで対応検討
- GUI ファイルマネージャーとの統合
- リモートホストへのディレクトリジャンプ (SSH 連携)

## 3. 機能要件

### 3.1 ファジー名前ジャンプ (fuzzy jump)

ディレクトリ名の一部を入力するだけで対象ディレクトリへジャンプする機能。

| ID | 要件 | 詳細 |
|----|------|------|
| F-1.1 | 部分一致ジャンプ | 入力文字列をディレクトリ名に部分一致させ、候補が1件の場合は即座に移動する |
| F-1.2 | 複数候補のインタラクティブ選択 | 候補が複数の場合、インクリメンタルサーチUI（fzf 等）で選択させる |
| F-1.3 | サブディレクトリの再帰検索 | カレントディレクトリ配下を再帰的に検索する（深さ上限: デフォルト 5） |
| F-1.4 | グローバル検索モード | オプション `-g` 指定時はホームディレクトリ以下を対象とする |
| F-1.5 | 検索除外パターン設定 | `node_modules` / `.git` / `dist` 等を設定ファイルで除外指定できる |

```bash
$ cd proj       # ~/dev/my-project にジャンプ
$ cd -g conf    # ホーム配下の conf ディレクトリを全検索
```

---

### 3.2 ブックマーク管理 (bookmark)

頻繁に使うディレクトリをエイリアス付きで登録・利用できる機能。

| ID | 要件 | 詳細 |
|----|------|------|
| F-2.1 | ブックマーク追加 | `cd -a [名前]` でカレントディレクトリをブックマークに登録する |
| F-2.2 | ブックマークへのジャンプ | `cd @[名前]` またはエイリアス名のみでジャンプする |
| F-2.3 | ブックマーク一覧表示 | `cd -l` でブックマーク一覧を番号・名前・パス形式で表示する |
| F-2.4 | ブックマーク削除 | `cd -d [名前]` で指定ブックマークを削除する |
| F-2.5 | ブックマーク編集 | `cd -e` でブックマークファイルを `$EDITOR` で開く |
| F-2.6 | ブックマーク名補完 | bash / zsh の Tab 補完でブックマーク名を補完する |
| F-2.7 | 存在確認 | ジャンプ時にパスが存在しない場合は警告を表示して移動しない |

```bash
$ cd -a myproj        # 現在地を 'myproj' としてブックマーク
$ cd @myproj          # ブックマーク 'myproj' へジャンプ
$ cd -l               # 一覧表示
```

---

### 3.3 ディレクトリ履歴管理 (history)

訪問済みディレクトリの履歴を自動記録し、素早く戻れる機能。

| ID | 要件 | 詳細 |
|----|------|------|
| F-3.1 | 履歴自動記録 | `cd` 実行のたびに訪問パスを timestamp 付きで記録する |
| F-3.2 | 履歴一覧表示 | `cd -H` で訪問頻度順 or 時系列で履歴を表示する |
| F-3.3 | 履歴ジャンプ | `cd -[番号]` で履歴 N 番目のディレクトリへジャンプする |
| F-3.4 | インタラクティブ選択 | `cd -H` をオプションなしで実行すると fzf UI で選択できる |
| F-3.5 | 重み付きランキング | 訪問頻度・最終訪問日時を加味した frecency スコアで並べ替える |
| F-3.6 | 履歴サイズ上限 | 保持する履歴件数の上限をコンフィグで設定できる (デフォルト: 1000件) |
| F-3.7 | 履歴クリア | `cd --clear-history` で全履歴を削除できる |

---

### 3.4 スタック操作 (pushd / popd 互換)

ディレクトリスタックを利用した前後移動機能。

| ID | 要件 | 詳細 |
|----|------|------|
| F-4.1 | スタックへのプッシュ | `cd -p [パス]` でスタックに積みながら移動する (pushd 相当) |
| F-4.2 | スタックからのポップ | `cd --` でスタックの前のディレクトリに戻る (popd 相当) |
| F-4.3 | スタック一覧 | `cd -s` でスタック全体を表示する |

| ID | 要件 | 詳細 |
|----|------|------|

## 5. コマンドインターフェース仕様

### 5.1 コマンド体系

| コマンド | 説明 |
|----------|------|
| `cd [名前/パス]` | ファジー検索でジャンプ（標準動作） |
| `cd -g [名前]` | ホーム以下をグローバル検索してジャンプ |
| `cd -a [名前]` | カレントをブックマークに追加 |
| `cd @[名前]` | ブックマークへジャンプ |
| `cd -l` | ブックマーク一覧表示 |
| `cd -d [名前]` | ブックマーク削除 |
| `cd -H` | 履歴インタラクティブ選択 |
| `cd -[N]` | 履歴 N 番目へジャンプ (例: `cd -1`) |
| `cd -p [パス]` | スタックに積んで移動 (pushd) |
| `cd --` | スタックを戻る (popd) |
| `cd -s` | スタック一覧表示 |
| `cd --clear-history` | 全履歴を削除 |
| `cd --config` | 設定ファイルを `$EDITOR` で開く |
| `cd --version` | バージョン情報表示 |
| `cd --help` | ヘルプ表示 |

### 5.2 設定ファイルスキーマ (config.toml)

```toml
[search]
max_depth        = 5
global_root      = "~"
exclude_patterns = ["node_modules", ".git", "dist", ".cache"]

[history]
max_entries = 1000
sort        = "frecency"   # frecency | time | alpha

[ui]
color        = true
fuzzy_finder = "fzf"       # fzf | peco | internal
```

### 7.4 シェル連携アーキテクチャ

`cd` はシェルビルトインであるため、Go バイナリ単体ではカレントディレクトリを変更できない。以下の構成で対応する。

```
┌─────────────────────────────────┐
│  shell (bash / zsh)             │
│  function cd() {                │
│    local result                 │
│    result=$(scd "$@")  ← stdout にパスを出力
│    builtin cd "$result"         │
│  }                              │
└─────────────────────────────────┘
         ↕ exec
┌─────────────────────────────────┐
│  scd (Go バイナリ)              │
│  - 検索 / ブックマーク / 履歴   │
│  - 移動先パスを stdout に出力   │
│  - UI・エラーは stderr に出力   │
└─────────────────────────────────┘
```

- Go バイナリ (`scd`) はロジックを全て担い、**移動先パスを stdout に出力**する
- シェルの薄いラッパー関数 `cd()` が `scd` を呼び出し、受け取ったパスで `builtin cd` を実行する
- シェルスクリプトは init ファイル 1 枚 (`init.sh` / `init.zsh`) のみ、100行以内に収める

### 7.5 インストール手順

```bash
# バイナリインストール (推奨)
brew install example/tap/smart-cd        # macOS
# または
go install github.com/example/smart-cd/cmd/scd@latest

# シェル連携の有効化
echo 'eval "$(scd --init bash)"' >> ~/.bashrc   # bash
echo 'eval "$(scd --init zsh)"'  >> ~/.zshrc    # zsh
source ~/.bashrc
```

## 10. 用語集

| 用語 | 定義 |
|------|------|
| ファジーマッチ | 入力文字列とディレクトリ名の部分一致・あいまい一致検索 |
| ブックマーク | 任意の名前を付けて保存したディレクトリパス |
| frecency | frequency（訪問頻度）と recency（最近度）を合算したスコアリング手法 |
| スタック | pushd/popd で使う LIFO (後入れ先出し) のディレクトリ履歴 |
| フォールバック | エラー時に標準の `cd` コマンドへ処理を委譲すること |
